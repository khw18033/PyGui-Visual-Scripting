# Go1 카메라 연동 실패 원인 및 최종 해결 로그

작성일: 2026-03-27

## 1) 최종 결론
현재 카메라 연동이 성공한 핵심 이유는 아래 3가지다.

1. 네트워크 경로를 실제 게이트웨이(라즈베리파이) IP에 맞게 수정했다.
2. 나노(카메라 장치) 접근을 점프 호스트 경유로 고정했다.
3. 카메라 시작 단계를 복잡한 혼합 명령 대신, "정리(킬)"와 "송출 시작"으로 분리했다.

## 2) 주요 실패 원인 분석

### 원인 A. 라우팅 게이트웨이 IP 불일치
- 초기 라우트가 잘못된 게이트웨이 IP를 가리켜 노트북에서 192.168.123.x 대역으로 접근이 실패함.
- 현상:
  - ping/ssh 시 No route to host
  - ARP incomplete
- 조치:
  - raspberrypi.local의 실제 IP 확인 후 라우트 재등록.

### 원인 B. SSH 비대화식 실행과 sudo 프롬프트 충돌
- 백그라운드 스레드에서 SSH 실행 시 비밀번호 입력 프롬프트가 뜨면 자동화가 깨짐.
- 현상:
  - [sudo] password for unitree
  - 카메라 시작 로그는 진행되지만 실제 송출 미실행
- 조치:
  - SSH 키 인증 설정
  - sudo 호출 방식 정리

### 원인 C. 시작 단계에서 과도한 정리 명령으로 SSH 세션 불안정
- start 단계에 fuser/pkill/sudo 정리 명령이 섞이면 일부 환경에서 세션이 끊김.
- 현상:
  - rc=255, stderr/stdout 없음
- 조치:
  - 시작 단계를 분리
    - 1) 강제 종료(정리) 명령
    - 2) 송출 스크립트 실행 명령
  - 최종적으로 점프 호스트 경유 SSH로 안정화

### 원인 D. 수신 파이프라인 코덱/플러그인 불일치
- H264 경로에서 avdec_h264 요소 미설치 환경 존재.
- 현상:
  - no element "avdec_h264"
  - not-negotiated
  - receiver 프로세스 즉시 종료
- 조치:
  - 수신 파이프라인을 단순화
  - 현재 성공 코드는 JPEG RTP 수신 경로를 직접 사용

### 원인 E. STOP 단계에서 원격 kill 로직이 불필요하게 복잡
- 원격 stop 로직이 다시 세션 불안정 요소를 만들 수 있음.
- 조치:
  - 현재 코드는 로컬 수신기 종료 중심으로 동작
  - 원격 kill 블록은 주석 처리하여 안정성 우선

## 3) 성공한 현재 코드의 핵심 포인트

### START 단계
- 나노 접근 방식:
  - ssh -J pi@192.168.50.41 unitree@192.168.123.13
- 처리 순서:
  1. 원격 정리 명령 실행
  2. go1_send_both.sh 백그라운드 실행
  3. 로컬 gst 수신기 실행
- 로그 기준 정상 시퀀스:
  - Cam START Step 1
  - SSH commands sent successfully
  - Cam START Step 2
  - Cam START Step 3
  - Receiver listening on port 9400
  - Front camera stream is now Running

### 수신 단계
- 로컬에서 UDP 9400을 받아 바로 파일 저장.
- front_000001.jpg 형태로 생성되면 정상.

### STOP 단계
- 로컬 gst 수신기 종료 중심으로 처리.

## 4) 재발 방지 체크리스트

1. 라우팅
- raspberrypi.local 해석 IP와 라우트 게이트웨이 일치 여부 확인.

2. SSH
- 노트북 -> 라즈베리파이 -> 나노 점프 경유 연결 확인.
- 키 인증 유효성 확인.

3. 나노 송출 스크립트
- /home/unitree/go1_send_both.sh 존재/실행 권한 확인.

4. 수신기
- 노트북에서 UDP 9400 패킷 유입 확인.
- /dev/shm 대상 폴더 권한 및 파일 생성 확인.

5. 코덱/플러그인
- 필요한 gstreamer 요소 존재 여부 사전 점검.

## 5) 요약
초기 실패는 단일 원인이 아니라 네트워크 경로, SSH 인증/실행 방식, 수신 파이프라인 요소가 겹쳐 발생했다. 현재 성공 코드는 경로를 바로잡고, SSH를 점프 호스트 기반으로 고정하며, 카메라 시작 절차를 분리해 안정화한 상태다.
