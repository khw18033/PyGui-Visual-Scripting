



### 🛠️ 리팩토링 로드맵: 견고한 아키텍처로의 전환

#### 1. 코드 분할 및 Git 전략 도입 (가장 시급함)

1,400줄이 넘어가는 단일 파일은 유지보수의 한계점에 도달했어. 도메인별로 파일을 나누는 모듈화가 필수야.

* **파일 분할 구조 (예시):**
* `main.py`: 프로그램 실행 진입점 및 메인 루프.
* `core/engine.py`: 노드 실행(Execution) 엔진.
* `ui/dpg_manager.py`: DPG 화면 렌더링 및 UI 전담.
* `nodes/base.py`: 노드 추상 클래스.
* `nodes/robots/mt4.py`, `nodes/robots/go1.py`, `nodes/robots/ep01.py`: 각 로봇별 노드 로직.


* **💡 교수님 조언 적용 (Git Flow):**
파일 분할을 시작하기 전에 **반드시 파일명에 버전(v11, v12)을 붙이는 방식을 버려야 해.** Git Repository를 제대로 초기화하고, `main` 브랜치는 항상 돌아가는 버전으로 둔 채로 `refactor/code-split` 같은 새로운 브랜치를 파서 작업한 뒤 Merge 하는 현대적인 방식으로 넘어가야 해.

#### 2. DPG 의존성 제거 및 UI 스키마 개편 (OCP 원칙)

이 부분이 이번 리팩토링의 핵심(개방-폐쇄 원칙)이야. 노드 클래스 내부에서 DPG 관련 코드를 완전히 덜어내야 해.

* **UI 분리:** `BaseNode`나 하위 로봇 노드에서 `build_ui()` 함수를 아예 삭제해. 노드는 오직 **"입력값 연산"**과 **"출력값 생성"**만 담당하는 순수 파이썬 로직 덩어리로 만들어야 해.
* **💡 교수님 조언 적용 (List 스키마):**
노드가 UI에게 "나 이런 입력창 필요해"라고 알려줄 때, 기존처럼 Dictionary(`{'x': 200, 'y': 0}`)를 쓰지 말고, 순서가 보장되는 **List나 Array**(`[("X", 200), ("Y", 0)]`)를 반환하도록 바꿔야 해.
* **UI 매니저 도입:** 새로 만들 `UIManager` 클래스가 각 노드의 List 스키마를 읽어와서 DPG 에디트 박스나 버튼을 대신 그려주는(Client) 역할을 맡게 될 거야.

#### 3. 카메라 노드의 단일 책임 원칙(SRP) 적용

`FileSaveCamera` 노드가 영상 획득, 화면 자르기(Calibration), ArUco 탐지, 파일 저장까지 너무 많은 일을 하고 있어.

* **노드 쪼개기:** 레고 블록처럼 조립할 수 있게 기능을 잘게 쪼개야 해.
1. **`Video Source Node`**: 카메라 영상을 읽어와서 메모리(데이터)로만 내보냄.
2. **`Image Save Node`**: 영상을 받아 폴더에 저장함.
3. **`UDP Broadcast Node`**: 영상을 받아 외부 망으로 쏴줌 (교수님이 언급하신 브로드캐스팅 기능).


* 이렇게 분리하면 "저장 없이 브로드캐스팅만 하는 노드 트리"를 GUI에서 자유롭게 마우스로 연결해서 만들 수 있게 돼.

#### 4. 상태 및 데이터 기반 흐름(Data Flow)으로 엔진 개조

현재 노드 실행 함수(`execute_graph_once`)는 하얀색 Flow 선을 따라 1->2->3 순서대로 강제 실행되는 '동기형(Synchronous) 절차 지향'에 가까워.

* **데이터 파이프라인으로 전환:** 노드가 실행되는 조건을 "이전 노드가 끝났는가?"가 아니라 **"내 입력 핀에 필요한 데이터(Data 선)가 모두 들어왔는가?"**로 바꿔야 해.
* **💡 교수님 조언 적용 (Enum 및 Input 분리):**
* "Offline", "Connecting" 같은 문자열을 비교하며 상태를 체크하지 말고, `Enum(0: OFFLINE, 1: IDLE, 2: RUNNING)`과 비트 연산을 써서 엔진이 빠르게 계산하게 만들어야 해.
* 키보드 노드(`MT4KeyboardNode`) 안에 있는 `dpg.is_key_down` 같은 함수도 빼내서, 외부의 `InputManager`가 비동기적으로 키 상태를 업데이트하고 노드는 그 데이터만 읽어오게 수정해야 해.
